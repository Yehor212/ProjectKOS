# GAME DESIGN BIBLE — ProjectKOS

> **Закон логіки і логічності (The Logic Law)**
> Кожна гра — це замкнутий автомат з чіткими правилами. Якщо правило неможливо пояснити 3-річній дитині за 5 секунд анімації — правило зламане.

## 1. shadow_match.gd (Shadow Match)

**game_id:** `shadow_match` | **Механіка:** Drag-to-silhouette | **Раунди:** 5 | **Вік:** TODDLER

### Концепт
Гравець бачить ряд чорних силуетів тварин і одну кольорову тварину внизу екрану. Потрібно перетягнути кольорову тварину на силует, якому вона відповідає. Розвиває впізнавання форм та просторове мислення; дитину приваблює контраст між яскравою тваринкою та таємничими тінями.

### Алгоритм гравця
1. Побачити кольорову тварину (знизу) і ряд чорних силуетів (зверху).
2. Ідентифікувати силует, що збігається з формою тварини.
3. Перетягнути тварину на правильний силует (magnetic assist для тоддлерів).
4. При попаданні: squish-bounce анімація, звук `success`, конфеті. Раунд +1.
5. При помилці: силует трясе, тварина повертається на місце (без `error` SFX для тоддлерів — тільки `click`).
6. Після `MAX_ROUNDS=5` правильних раундів — фінал з `_calculate_stars`.

### Контент-пул
- **Унікальних варіантів**: 19 (усі записи `GameData.ANIMALS_AND_FOOD`)
- **Вичерпується після**: ~4-5 сесій (кожен раунд — 1 нова тварина; `_used_indices` скидається після вичерпання)
- **Рандомізація**: вибір 3-4 тварин на раунд, рандомний `_target_id`, позиції силуетів рівномірні

### Replay Hook
- Щоразу нова комбінація тварин і нова "загадкова" тварина для перетягування
- `_used_indices` гарантує різних тварин у межах сесії
- Preschool: "швидше без помилок = більше зірок"

### Складність по раундах
| Раунд | Параметр | Toddler | Preschool |
|-------|----------|---------|-----------|
| 1 | Силуетів | 3 | 3 |
| 3 | Силуетів | ~3-4 | ~3-4 |
| 5 | Силуетів | 4 | 4 |

`_scale_by_round_i(3, 4, _current_round, MAX_ROUNDS)`. Toddler має `magnetic_assist = true` та `snap_radius_override`.

### Дорожня карта покращень
- [v1.1] Розширити пул до 30+ тварин; поточний hard-cap 4 силуети
- [v1.2] Preschool: силуети частково перекривають один одного
- [v1.3] "Mirror mode" — дзеркальні силуети

| A1 | ✅ | `get_tutorial_demo()` — drag demo |
| A2 | ✅ | `MAX_ROUNDS=5`, немає deadlock |
| A3 | ⚠️ | `magnetic_assist` для тоддлера, але `_slots_per_round` однаковий |

## 2. shape_sorter.gd (Shape Sorter)

**game_id:** `shape_sorter` | **Механіка:** Drag-to-slot | **Раунди:** T:3 / P:1 | **Вік:** ALL

### Концепт
**Toddler**: перетягнути геометричні фігури (коло, квадрат, трикутник, ромб) у відповідні отвори-слоти. **Preschool**: зібрати ракету з 4 деталей (корпус, ніс, два крила). Toddler розвиває розпізнавання форм; Preschool — просторове мислення та асемблювання.

### Алгоритм гравця
**Toddler:** Бачить фігури внизу + слоти зверху → drag → після всіх → конфеті → наступний раунд → 3 раунди → `_finish()`.
**Preschool:** 4 деталі ракети → drag кожну → всі на місці → blast-off анімація → `_finish()`.

### Контент-пул
- **Toddler**: 4 фігури в `TODDLER_SHAPES_POOL`; кожен раунд 2-4 з них
- **Preschool**: 4 деталі ракети (фіксовано) — контент однаковий кожен раз
- **Вичерпується після**: Toddler — не вичерпується; Preschool — 1 сесія (лише ракета)
- **Рандомізація**: Toddler: shuffle + кількість; Preschool: shuffle позицій деталей

### Replay Hook
- Toddler: рандомний порядок та вибір фігур
- Preschool: blast-off анімація ракети — емоційний момент
- Змагання "скільки помилок зробив?"

### Складність по раундах
| Раунд | Toddler | Preschool |
|-------|---------|-----------|
| 0 | 2 фігури | 4 деталі (фікс) |
| 1 | 3 фігури | — |
| 2 | 4 фігури | — |

`_scale_by_round_i(2, 4, _round, 3)`.

### Дорожня карта покращень
- [v1.1] Preschool: 2-3 різних "конструктори" (корабель, будиночок, машина)
- [v1.2] Toddler: звуки для кожної фігури
- [v1.3] Preschool: анімований персонаж-інструктор

## 3. counting_game.gd (Count with Tofie)

**game_id:** `counting` | **Механіка:** Drag (T) / Tap (P) | **Раунди:** 5 | **Вік:** ALL

### Концепт
**Toddler**: перетягнути рівно N фруктів у кошик (з відволікачами). **Preschool**: розв'язати A+B=? і натиснути відповідь з 3 варіантів. Розвиває кількісне мислення та базову арифметику.

### Алгоритм гравця
**Toddler:** Число N + кошик + N+3 фруктів → drag правильних → кожен → coin SFX → N зібрано → раунд done.
**Preschool:** Рівняння A+B=? + 3 варіанти → tap правильний → неправильні сіріють → 5 раундів.

### Контент-пул
- **Фруктів**: 5 типів (`FRUITS`); **Toddler відволікачі**: 3 (`TODDLER_DISTRACTORS`)
- **Вичерпується після**: не вичерпується (рандомні рівняння/фрукти)
- **Рандомізація**: рандомний фрукт-ціль, рандомні A/B, shuffle позицій

### Replay Hook
- Toddler: різний фрукт і кількість; pitch-bend монеток
- Preschool: рівняння завжди нові; 5 зірок при 0 помилках

### Складність по раундах
| Раунд | Toddler (кількість) | Preschool (A max / B max) |
|-------|---------------------|--------------------------|
| 1 | 1-3 | 3 / 2 |
| 5 | 3-5 | 6 / 4 |

### Дорожня карта покращень

## 4. color_pop.gd (Color Pop)

**game_id:** `color_pop` | **Механіка:** Tap (time attack) | **Раунди:** 1 (45с) | **Вік:** ALL

### Концепт
Мильні бульбашки піднімаються 45 секунд. **Toddler**: лопай будь-яку (+1). **Preschool**: лопай тільки цільового кольору (змінюється кожні 10с), неправильний — штраф. Розвиває кольоророзрізнення та реакцію.

### Алгоритм гравця
**T:** Тап будь-якої бульбашки → тваринка-нагорода → кожні 5 попів `_register_correct()` → 45с → фініш.
**P:** Цільовий колір зверху → тільки правильного кольору (+2 очки) → неправильний = error → кожні 10с зміна кольору → 45с → фініш.

### Контент-пул
- **Кольорів**: 5 (`COLORS`: red, blue, green, yellow, purple)
- **Тварин-нагород (T)**: 19; **Spawn**: interval 1.5→0.9с (T), 0.7→0.42с (P)
- **Вичерпується після**: ніколи
- **Рандомізація**: X-позиція, колір, швидкість, тварина-нагорода

### Replay Hook
- Score — personal best; тваринки-сюрпризи при попах
- Preschool: рейтинг точності через зірки

### Складність по раундах
| Час | T spawn | P spawn | Швидкість |
|-----|---------|---------|-----------|
| 0с | 1.5с | 0.7с | ×1.0 |
| 45с | 0.9с | 0.42с | ×1.4 |

`lerpf(base, base * 0.6, progress)`.

### Дорожня карта покращень

## 5. memory_cards.gd (Memory Cards)

**game_id:** `memory` | **Механіка:** Tap-to-flip/highlight | **Раунди:** T:3 / P:2 | **Вік:** ALL

### Концепт
**Toddler**: картки ВІДКРИТІ — тапай пари. **Preschool**: класичне мемо — картки перевернуті, flip-анімація, `MISMATCH_PAUSE=1.5с`. Розвиває короткочасну пам'ять.

### Алгоритм гравця
**T:** Всі відкриті → тап першої (highlight) → тап другої → пара = залишаються → всі пари done → раунд.
**P:** Всі перевернуті → тап flip → другий тап → порівняння → пара = `set_matched()`, не пара = flip back після 1.5с.

### Контент-пул
- **Унікальних пар**: 19 тварин; **Пар**: T: 3→4, P: 3→6
- **Grids**: T: `3×2`, `3×2`, `4×2`; P: `3×2`, `4×3`
- **Вичерпується після**: `_used_indices` запобігає повторенню
- **Рандомізація**: вибір пар, shuffle карток

### Replay Hook
- P: мінімум помилок = 5 зірок; T: "знайди всіх тваринок"
- Victory dance всіх карток

### Складність по раундах

|-------|---------|-----------|
| 1 | 3×2 (3 пари) | 3×2 (3 пари) |
| 2 | 3×2 | 4×3 (6 пар) |

### Дорожня карта покращень

## 6. food_game.gd (Hungry Pets)

**game_id:** `hungry_pets` | **Механіка:** Drag food to animal | **Раунди:** dynamic (3→5 пар) | **Вік:** TODDLER

### Концепт
Головна гра проєкту: перетягнути правильну їжу до відповідної тварини. Тварини зверху, їжа знизу. Правильне → happy-анімація, неправильне → sad-анімація. Розвиває асоціативне мислення.

### Алгоритм гравця
1. N тварин (3-5) + N видів їжі (перемішані).
2. Drag їжу до тварини. Правильно: `play_happy()`, `unlock_animal()`, combo pitch-bend.
3. Неправильно: `play_sad()`, `click`, їжа повертається.
4. `RoundManager` додає нові пари → після всіх 19 → `game_won`.

### Контент-пул
- **Унікальних пар**: 19 (`GameData.ANIMALS_AND_FOOD`)
- **Пар одночасно**: 3 (0-2 rounds) → 4 (3-6) → 5 (7+)
- **Вичерпується після**: ~4-6 сесій
- **Рандомізація**: `selected_indices` shuffled, позиції рандомні

### Replay Hook
- `ProgressManager.unlock_animal()` — колекція
- Combo pitch-bend; хмаринки-декорації; тварини "засипають"

### Складність по раундах
| rounds_played | Пар одночасно |
|---------------|--------------|
| 0-2 | 3 |
| 3-6 | 4 |
| 7+ | 5 |

### Дорожня карта покращень
- [v1.1] Preschool режим: тварина прихована (силует), вгадати за їжею

## УНІВЕРСАЛЬНИЙ FLOW

```
[LAUNCH] → BaseMiniGame._ready()
    ├─ UI overlay (back, pause, stars)
    ├─ Exit confirmation overlay
    ├─ Tutorial system setup
    └─ Game-specific _ready()

[ROUND START] → _start_round()
    ├─ Clear previous round data
    ├─ Calculate difficulty (_scale_by_round)
    ├─ Spawn items/targets
    ├─ Deal animation (staggered fade-in)
    └─ Unlock input after animation

[GAMEPLAY LOOP]
    ├─ Player interacts (drag/tap/draw)
    ├─ [CORRECT] → _register_correct(), success sfx, particles
    ├─ [WRONG T] → click sfx, wobble/snap, no penalty
    ├─ [WRONG P] → _errors++, _register_error(), error sfx+smoke
    ├─ [IDLE 5s] → pulse hint (level 0)
    ├─ [IDLE 10s] → stronger pulse (level 1)
    ├─ [IDLE 15s] → tutorial hand (level 2)
    ├─ [SCAFFOLD] → 2(T)/3(P) consecutive errors → show answer
    └─ [ROUND COMPLETE] → confetti, delay, next round

[GAME FINISH] → finish_game()
    ├─ _calculate_stars(penalty) — CENTRALIZED
    ├─ Save progress (ProgressManager)
    ├─ Log analytics
    └─ Show level_complete_overlay
```
